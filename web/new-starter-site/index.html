HTML

An HTML file tells the browser how to format the content of a web page. 1 HTML file specifies the format of one web page. HTML files consist of tags and text. The tags specify how the text of the web page should be formatted. The top line of text for this web page is inside "heading 1" tags (<h1>), the next line is inside "heading 2" tags (<h2>), and this text is inside "paragraph" tags (<p>).

Tags & Elements

Almost all HTML tags come in pairs. That is, for every opening tag (i.e., <p>), there is also a closing tag (i.e., </p>). The closing tag is indicated by a preceding slash. The text between the opening and closing tag is what gets formatted, and what the end users see. Typically, one HTML element consists of an opening tag, a closing tag, and the text in between.

Some elements don't have closing tags. These are called void elements. Some examples of void elements include forms of input like buttons and textfields (<input>), line breaks (<br>), and links to CSS/other files (<link>). Void elements only have an opening tag (no following text or closing tag). If a void element contains text (e.g., a button with the word 'Submit' on it), it is specified via an attribute.

Another way to classify elements is whether they are block or inline. Block elements appear on their own line. By default, browsers will add space between block elements. Inline elements appear on the same line. Although most elements are block, you can adjust the CSS property 'display' to block or inline.

Attributes

Opening HTML tags can define attributes to format the element.

Syntax:    <p attribute_name='attribute_value'>

Example: <input type='submit' value='click me to do nothing'>

Special attributes

  • class is a useful attribute that allows several elements to share the same format. In conjunction with CSS, classes enable a web page to look appealing.
  • id is a useful attribute that uniquely identifies an element. id's can be included in a hypertext reference to link to a specific part of a web page. It is also used by Javascript to give elements interactivity.

ex:

<div id="top">...</div>

<a href=#top>Back to Top</a>

Inheritance

When tags are nested, the inner tag inherits attributes from the outer tag.

ex:

<div class="blue-text">

<h4>Heading</h4>

<p>This paragraph inherits blue text</p>

</div>

Common HTML elements (with relevant attributes)

Common HTML elements

  • <p> paragraph
  • <h1> heading 1: this is the one that search engines look for. Should be one per HTML file
  • <a> anchor: specify a hypertext reference to another web page. Can be a hyperlink to a different webiste (URL) or a hyperlink to a page on the same site (filepath)
  • <div> division: used to divide the document into sections. Convenient way to give certain elements shared properties (see inheritance). div is handy for setting default styling within a section, but the most important function of div is layout positioning (separating header, navigation bar, main, content, etc.).
  • <form> form: create a form to collect user input.
    • action: specifies the backend program that will process the information when the form is submitted. This program is stored & runs on the server. Common backend processing languages include PHP, Ruby, Python, Perl, Java, & C#.
    • method: specifies the HTTP method to use when sending form-data. Default method is 'get', can also be set to 'post'. Essentially GET is used to retrieve remote data, and POST is used to insert/update remote data. Most forms will use post.

Common void elements

  • <input> input: used to retreive user input inside a form.
    • type: the type of input to use.
      • type='text': a textfield
      • type='password': a password textfield
    • name: the variable name that the backend program (specified in <form action='...' method='post'>) will use to reference this data
  • <br> break
  • <img> image: an image. You can make an image a link to another web page by nesting it inside an anchor tags: <a><img></a>
    • src: the filepath of the image
    • alt: the alternate text, what the user sees if the image doesn't load.
